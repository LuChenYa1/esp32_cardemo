# 舵机自动往复控制任务 (Servo Task)

## 元信息

- **版本**: 1.0.0
- **更新日期**: 2024-12-20
- **维护者**: 项目团队
- **使用状态**: main.c使用中

---

## 组件简介

舵机任务模块实现了RS485舵机的自动往复控制功能。该模块作为FreeRTOS任务运行，每3秒在两组预定义位置之间切换，控制4个舵机（ID 0x02-0x05）实现自动巡视功能。

### 主要特性

- 自动往复运动：每3秒在两组位置之间自动切换
- 多舵机控制：同时控制4个舵机（ID 0x02, 0x03, 0x04, 0x05）
- 错误重试机制：通信失败时自动重试最多3次
- 非阻塞设计：作为独立FreeRTOS任务运行，不影响其他功能
- 可配置位置组：支持自定义舵机位置组合

### 适用场景

适用于需要舵机自动巡视、往复运动的应用场景，如摄像头云台自动扫描、机械臂自动演示等。

## 硬件连接

### 引脚分配

| 功能 | GPIO引脚 | 接口标识 | 说明 |
|------|---------|---------|------|
| UART TX | GPIO1 | - | RS485发送数据 |
| UART RX | GPIO3 | - | RS485接收数据 |
| 方向控制 | GPIO19 | - | RS485方向控制（TX/RX切换）|

### 接线说明

1. 将RS485转换模块的TX连接到ESP32的GPIO1
2. 将RS485转换模块的RX连接到ESP32的GPIO3
3. 将RS485转换模块的DE/RE引脚连接到ESP32的GPIO19
4. RS485总线连接到舵机的A/B端子
5. 确保所有设备共地（GND连接）

### 注意事项

- ⚠️ 使用UART0前需要关闭ESP_LOG输出到UART0，避免日志干扰通信
- ⚠️ RS485总线需要120Ω终端电阻（通常在总线两端）
- ⚠️ 舵机需要独立供电（通常6-12V），ESP32仅提供控制信号
- ⚠️ 确保RS485转换模块与ESP32共地

### 通信参数

| 参数名称 | 值 | 说明 |
|---------|-----|------|
| 波特率 | 1000000 bps | 1Mbps高速通信 |
| 数据位 | 8位 | 标准配置 |
| 停止位 | 1位 | 标准配置 |
| 校验位 | 无 | 无校验 |
| 协议 | RS485串行舵机协议 | 厂商自定义协议 |

## 功能说明

### 核心功能

#### 功能1：自动往复运动控制

任务以3秒为周期，在两组预定义位置之间自动切换：
- **位置组1**：舵机ID 0x02=800, 0x03=650, 0x04=850, 0x05=750
- **位置组2**：舵机ID 0x02=350, 0x03=750, 0x04=750, 0x05=680

时序说明：
```
0s  -> 位置组1 (800, 650, 850, 750)
3s  -> 位置组2 (350, 750, 750, 680)
6s  -> 位置组1 (800, 650, 850, 750)
9s  -> 位置组2 (350, 750, 750, 680)
... -> 循环往复
```

#### 功能2：通信错误重试机制

当舵机通信失败时，自动重试机制：
1. 每个舵机命令最多重试3次
2. 每次重试间隔50ms
3. 记录错误日志但继续执行其他舵机
4. 不会因单个舵机失败而阻塞整个任务

#### 功能3：FreeRTOS任务管理

作为独立的FreeRTOS任务运行：
- 任务优先级：3（低于实时控制任务）
- 栈大小：4KB
- 非阻塞设计，不影响其他任务
- 支持任务创建和删除

### 配置参数

| 参数名称 | 默认值 | 取值范围 | 说明 |
|---------|--------|---------|------|
| 切换周期 | 3000ms | 1000-10000ms | 位置组切换间隔 |
| 舵机速度 | 50 | 0-255 | 舵机运动速度（0最快）|
| 重试次数 | 3次 | 1-10次 | 通信失败重试次数 |
| 重试间隔 | 50ms | 10-500ms | 每次重试的等待时间 |
| 任务优先级 | 3 | 1-10 | FreeRTOS任务优先级 |
| 任务栈大小 | 4096字节 | 2048-8192字节 | 任务栈空间 |

## API接口

### 任务管理函数

```c
/**
 * @brief 创建舵机自动往复控制任务
 * 
 * 初始化RS485舵机通信，创建FreeRTOS任务实现舵机自动往复运动
 * 任务优先级设置为3，每3秒在两组位置之间切换
 * 
 * @return TaskHandle_t 返回创建的任务句柄，失败返回NULL
 */
TaskHandle_t servo_task_create(void);
```

**参数说明**：
- 无参数

**返回值**：
- 成功：返回任务句柄（非NULL）
- 失败：返回NULL

**使用说明**：
- 自动初始化RS485舵机驱动
- 自动关闭ESP_LOG输出到UART0，避免日志干扰
- 任务创建后立即开始运行
- 只需调用一次

---

```c
/**
 * @brief 删除舵机任务并释放资源
 * 
 * 停止舵机任务并释放相关资源
 * 
 * @param task_handle 要删除的任务句柄
 * @return 无返回值
 */
void servo_task_delete(TaskHandle_t task_handle);
```

**参数说明**：
- `task_handle`: 由`servo_task_create()`返回的任务句柄

**返回值**：
- 无返回值

**使用说明**：
- 传入NULL会被忽略
- 删除后任务句柄失效，不可再次使用
- 删除任务不会自动关闭RS485驱动

---

### 位置控制函数

```c
/**
 * @brief 设置舵机位置组（带重试机制）
 * 
 * 向4个舵机发送位置命令，失败时自动重试最多3次
 * 
 * @param pos 舵机位置组指针
 * @return int 返回0表示成功，-1表示失败
 */
int servo_set_position_group_with_retry(const ServoPositionGroup_t *pos);
```

**参数说明**：
- `pos`: 舵机位置组结构体指针，包含4个舵机的目标位置

**返回值**：
- `0`: 所有舵机设置成功
- `-1`: 至少一个舵机设置失败

**使用说明**：
- 依次发送4个舵机的位置命令（ID 0x02-0x05）
- 每个舵机失败时自动重试3次
- 即使部分舵机失败，也会继续执行其他舵机
- 每个舵机命令发送后延时50ms

---

### 数据结构

```c
/**
 * @brief 舵机位置组结构体
 * 
 * 定义一组舵机的目标位置，用于实现往复运动
 */
typedef struct {
    uint16_t servo_id_02_pos;  // 舵机ID 0x02的位置 (0-1000)
    uint16_t servo_id_03_pos;  // 舵机ID 0x03的位置 (0-1000)
    uint16_t servo_id_04_pos;  // 舵机ID 0x04的位置 (0-1000)
    uint16_t servo_id_05_pos;  // 舵机ID 0x05的位置 (0-1000)
} ServoPositionGroup_t;
```

**字段说明**：
- 位置范围：0-1000（具体范围取决于舵机型号）
- 位置值越大，舵机转角越大
- 中间位置通常为500

## 使用示例

### 基本使用示例

```c
#include "servo_task.h"
#include "esp_log.h"

static const char *TAG = "app_main";

void app_main(void)
{
    // 创建舵机任务
    TaskHandle_t servo_handle = servo_task_create();
    
    if (servo_handle == NULL) {
        ESP_LOGE(TAG, "舵机任务创建失败");
        return;
    }
    
    ESP_LOGI(TAG, "舵机任务运行中，每3秒自动切换位置");
    
    // 任务会自动运行，无需额外操作
    // 主程序可以继续执行其他任务
    
    // 如需停止任务（可选）
    // vTaskDelay(pdMS_TO_TICKS(30000));  // 运行30秒后停止
    // servo_task_delete(servo_handle);
    // ESP_LOGI(TAG, "舵机任务已停止");
}
```

### 自定义位置组示例

```c
#include "servo_task.h"
#include "esp_log.h"

static const char *TAG = "custom_servo";

// 定义自定义位置组
static const ServoPositionGroup_t custom_position = {
    .servo_id_02_pos = 600,
    .servo_id_03_pos = 700,
    .servo_id_04_pos = 800,
    .servo_id_05_pos = 500
};

void custom_servo_control(void)
{
    // 初始化RS485舵机驱动
    servo485_init();
    
    // 手动设置舵机位置组
    int result = servo_set_position_group_with_retry(&custom_position);
    
    if (result == 0) {
        ESP_LOGI(TAG, "舵机位置设置成功");
    } else {
        ESP_LOGE(TAG, "舵机位置设置失败");
    }
}
```

### 任务生命周期管理示例

```c
#include "servo_task.h"
#include "freertos/FreeRTOS.h"
#include "freertos/task.h"
#include "esp_log.h"

static const char *TAG = "lifecycle";

void servo_lifecycle_demo(void)
{
    TaskHandle_t servo_handle = NULL;
    
    // 1. 创建任务
    ESP_LOGI(TAG, "创建舵机任务...");
    servo_handle = servo_task_create();
    
    if (servo_handle == NULL) {
        ESP_LOGE(TAG, "任务创建失败");
        return;
    }
    
    // 2. 运行30秒
    ESP_LOGI(TAG, "舵机任务运行中...");
    vTaskDelay(pdMS_TO_TICKS(30000));
    
    // 3. 删除任务
    ESP_LOGI(TAG, "停止舵机任务...");
    servo_task_delete(servo_handle);
    
    // 4. 等待5秒后重新创建
    vTaskDelay(pdMS_TO_TICKS(5000));
    ESP_LOGI(TAG, "重新创建舵机任务...");
    servo_handle = servo_task_create();
}
```

## 注意事项

### 硬件限制

- ⚠️ 使用UART0，与ESP_LOG默认输出冲突，需要关闭日志输出
- ⚠️ RS485总线需要120Ω终端电阻，否则可能通信不稳定
- ⚠️ 舵机需要独立供电，ESP32仅提供控制信号
- ⚠️ GPIO19用于RS485方向控制，不能复用于其他功能

### 软件限制

- ⚠️ 必须先初始化RS485舵机驱动（`servo_task_create()`会自动初始化）
- ⚠️ 位置组在代码中硬编码，修改需要重新编译
- ⚠️ 任务优先级为3，不适合实时性要求极高的场景
- ⚠️ 每个舵机命令发送后延时50ms，4个舵机总计约200ms

### 线程安全

- 任务内部使用FreeRTOS任务，线程安全
- 不建议在多个任务中同时调用`servo_set_position_group_with_retry()`
- RS485总线是半双工通信，同时发送会导致冲突
- 如需多任务控制，建议使用互斥锁或消息队列

### 性能考虑

- 每3秒切换一次位置，适合慢速巡视场景
- 如需更快切换，修改`vTaskDelay(pdMS_TO_TICKS(3000))`中的延时值
- 舵机速度参数为50，可调整范围0-255（0最快）
- 通信波特率1Mbps，单个命令约10ms完成

## 故障排除

### 常见问题

#### 问题1：舵机不动或动作异常

**现象**：调用`servo_task_create()`后，舵机没有反应或动作不正常

**原因**：
- RS485接线错误或接触不良
- 舵机供电不足或未供电
- 舵机ID配置错误
- UART0被ESP_LOG占用

**解决方案**：
1. 检查RS485接线：A/B线是否正确连接
2. 检查舵机供电：使用万用表测量电源电压
3. 确认舵机ID：使用舵机调试工具确认ID为0x02-0x05
4. 检查日志输出：确认`esp_log_level_set("*", ESP_LOG_NONE)`已执行
5. 使用示波器检查UART TX引脚是否有数据输出

#### 问题2：部分舵机工作，部分不工作

**现象**：4个舵机中只有部分舵机响应命令

**原因**：
- 某些舵机ID配置错误
- RS485总线信号质量差
- 舵机供电不足（电流不够）
- 舵机损坏或故障

**解决方案**：
1. 逐个测试舵机：单独连接每个舵机测试
2. 检查RS485总线：添加120Ω终端电阻
3. 增加供电电流：使用更大功率的电源
4. 查看错误日志：检查哪个舵机ID通信失败
5. 更换故障舵机

#### 问题3：任务创建失败

**现象**：`servo_task_create()`返回NULL

**原因**：
- 内存不足，无法创建任务
- 栈空间不足
- FreeRTOS配置错误

**解决方案**：
1. 检查可用内存：使用`esp_get_free_heap_size()`
2. 减少任务栈大小：修改`xTaskCreate()`中的栈大小参数
3. 增加FreeRTOS堆大小：修改`sdkconfig`中的配置
4. 删除不必要的任务，释放资源

#### 问题4：如何修改位置组？

**现象**：需要自定义舵机位置组

**原因**：位置组在代码中硬编码

**解决方案**：
1. 修改`servo_task.c`中的`position_group_1`和`position_group_2`
2. 位置范围通常为0-1000（具体取决于舵机型号）
3. 修改后重新编译代码
4. 或使用`servo_set_position_group_with_retry()`手动设置位置
