## 组件简介

语音控制模块通过UART1接收语音识别模块发送的命令，实现对摄像头识别模式、系统运行模式和速度参数的语音控制。该模块采用非阻塞式串口通信，确保命令处理不影响系统实时性。

### 主要特性

- 通过UART1接收语音命令（波特率115200）
- 支持7种摄像头识别模式切换（颜色、色块、人脸、二维码、数字、标签、20类）
- 支持摄像头亮度调节和复位控制
- 支持运行模式切换（巡线模式/命令模式）
- 支持速度档位切换（高速/低速）
- 非阻塞式命令处理，响应时间<10ms
- 使用原子变量保证多任务环境下的线程安全

### 适用场景

适用于需要语音控制的智能小车系统，特别是巡线小车、视觉识别小车等需要动态切换工作模式的应用场景。

## 硬件连接

### 引脚分配

| 功能 | GPIO引脚 | 接口标识 | 说明 |
|------|---------|---------|------|
| UART1 TX | GPIO35 | - | 发送数据到语音模块（保留） |
| UART1 RX | GPIO36 | - | 接收语音模块命令 |

### 接线说明

1. 将语音识别模块的TX引脚连接到ESP32的GPIO36（UART1 RX）
2. 将语音识别模块的RX引脚连接到ESP32的GPIO35（UART1 TX，可选）
3. 确保语音模块和ESP32共地（GND连接）
4. 语音模块供电电压根据模块规格（通常为3.3V或5V）

### 注意事项

- ⚠️ GPIO36为仅输入引脚，不能用作输出
- ⚠️ 确保语音模块波特率设置为115200
- ⚠️ 语音模块和ESP32必须共地，否则串口通信不稳定

## 功能说明

### 核心功能

#### 功能1：语音命令接收与解析

通过UART1非阻塞式接收语音模块发送的命令帧，命令帧格式为5字节：`[0xAA] [0x55] [CMD] [0x55] [0xAA]`。解析器会在接收缓冲区中查找有效的命令帧，提取命令字节并执行相应操作。

#### 功能2：摄像头识别模式控制

支持7种摄像头识别模式的语音切换：
- 颜色识别（0x01）：识别红、绿、蓝等颜色
- 色块识别（0x02）：识别特定颜色的色块
- 人脸识别（0x03）：检测人脸
- 二维码识别（0x04）：识别二维码内容
- 数字识别（0x05）：识别数字
- 标签识别（0x06）：识别预定义标签
- 20类物体识别（0x07）：识别20种常见物体

#### 功能3：摄像头亮度控制

支持通过语音命令调节摄像头亮度（0-4级），命令包括：
- 增加亮度（0x0C）
- 降低亮度（0x0D）

#### 功能4：运行模式控制

支持两种运行模式的切换：
- 巡线模式（0x11）：电机启动，小车按照灰度传感器进行巡线
- 命令模式（0x12）：电机停止，小车等待其他控制命令

#### 功能5：速度档位控制

支持两种速度档位的切换：
- 高速模式（0x0E）：基础速度999，PD参数Kp=8.0, Kd=12.0，后退时间16 ticks
- 低速模式（0x0F）：基础速度700，PD参数Kp=5.0, Kd=12.0，后退时间12 ticks

### 配置参数

| 参数名称 | 默认值 | 取值范围 | 说明 |
|---------|--------|---------|------|
| VOICE_UART_PORT | UART_NUM_1 | UART_NUM_0~2 | 使用的UART端口 |
| VOICE_UART_BAUD_RATE | 115200 | 9600~921600 | 串口波特率 |
| VOICE_TASK_PRIORITY | 4 | 1~24 | 任务优先级（低于Timer中断） |
| VOICE_TASK_STACK_SIZE | 8192 | 4096~16384 | 任务栈大小（字节） |
| VOICE_UART_RX_BUFFER_SIZE | 2056 | 256~4096 | 接收缓冲区大小 |

## API接口

### 初始化函数

```c
/**
 * @brief 初始化语音模块
 * 
 * @return ESP_OK 成功，ESP_FAIL 失败
 */
esp_err_t voice_module_init(void);
```

**参数说明**：
- 无

**返回值**：
- `ESP_OK`: 初始化成功
- `ESP_FAIL`: 初始化失败（UART配置失败或驱动安装失败）

**使用说明**：
在系统启动时调用，完成UART1配置、驱动安装和标志变量初始化。必须在创建任务前调用。

---

### 任务管理函数

```c
/**
 * @brief 创建语音模块UART接收任务
 * 
 * @return ESP_OK 成功，ESP_FAIL 失败
 */
esp_err_t voice_module_task_create(void);
```

**参数说明**：
- 无

**返回值**：
- `ESP_OK`: 任务创建成功
- `ESP_FAIL`: 任务创建失败

**使用说明**：
创建FreeRTOS任务，任务固定在Core 1上运行，避免影响Core 0的实时控制。任务会循环接收UART数据并解析命令。

```c
/**
 * @brief 删除语音模块任务
 */
void voice_module_task_delete(void);
```

**参数说明**：
- 无

**返回值**：
- 无

**使用说明**：
停止语音模块任务并释放资源。通常在系统关闭或重新配置时调用。

---

### 运行模式控制函数

```c
/**
 * @brief 获取运行模式标志（线程安全）
 * 
 * @return uint8_t 运行模式标志（0=命令模式，1=巡线模式）
 */
uint8_t voice_module_get_run_mode(void);
```

**参数说明**：
- 无

**返回值**：
- `0`: 命令模式（电机停止）
- `1`: 巡线模式（电机运行）

**使用说明**：
在电机控制逻辑中调用，判断是否应该驱动电机。使用原子操作保证线程安全。

```c
/**
 * @brief 设置运行模式标志（线程安全）
 * 
 * @param mode 运行模式（0=命令模式，1=巡线模式）
 */
void voice_module_set_run_mode(uint8_t mode);
```

**参数说明**：
- `mode`: 运行模式，0表示命令模式，1表示巡线模式

**返回值**：
- 无

**使用说明**：
可在其他模块中调用，手动设置运行模式。使用原子操作保证线程安全。

### 全局标志变量

```c
// 摄像头识别模式标志（使用原子变量）
extern _Atomic uint8_t Flag_Color;      // 颜色识别标志
extern _Atomic uint8_t Flag_Block;      // 色块识别标志
extern _Atomic uint8_t Flag_Face;       // 人脸识别标志
extern _Atomic uint8_t Flag_QRCODE;     // 二维码识别标志
extern _Atomic uint8_t Flag_NUMBER;     // 数字识别标志
extern _Atomic uint8_t Flag_LABEL;      // 标签识别标志
extern _Atomic uint8_t Flag_20CLASS;    // 20类识别标志

// 系统控制标志
extern _Atomic uint8_t Flag_Speed;      // 速度档位（1=低速，2=高速）
extern _Atomic uint8_t Flag_RunMode;    // 运行模式（0=命令模式，1=巡线模式）
```

**使用说明**：
这些全局标志变量使用原子类型（`_Atomic`），可以在多任务环境中安全访问。建议使用`atomic_load()`和`atomic_store()`函数进行读写操作。

## 使用示例

### 基本使用示例

```c
#include "voice_module.h"
#include "esp_log.h"

static const char *TAG = "app_main";

void app_main(void)
{
    // 1. 初始化语音模块
    esp_err_t ret = voice_module_init();
    if (ret != ESP_OK) {
        ESP_LOGE(TAG, "语音模块初始化失败");
        return;
    }
    ESP_LOGI(TAG, "语音模块初始化成功");
    
    // 2. 创建语音模块任务
    ret = voice_module_task_create();
    if (ret != ESP_OK) {
        ESP_LOGE(TAG, "语音模块任务创建失败");
        return;
    }
    ESP_LOGI(TAG, "语音模块任务创建成功");
    
    // 3. 在主循环中检查运行模式
    while (1) {
        uint8_t run_mode = voice_module_get_run_mode();
        if (run_mode == 1) {
            // 巡线模式：执行电机控制
            ESP_LOGI(TAG, "巡线模式运行中");
        } else {
            // 命令模式：停止电机
            ESP_LOGI(TAG, "命令模式，电机停止");
        }
        
        vTaskDelay(pdMS_TO_TICKS(1000));
    }
}
```

### 高级使用示例：访问识别模式标志

```c
#include "voice_module.h"
#include <stdatomic.h>

void check_camera_mode(void)
{
    // 使用原子操作读取标志变量
    uint8_t color_mode = atomic_load(&Flag_Color);
    uint8_t face_mode = atomic_load(&Flag_Face);
    uint8_t speed = atomic_load(&Flag_Speed);
    
    if (color_mode) {
        ESP_LOGI(TAG, "当前为颜色识别模式");
    }
    
    if (face_mode) {
        ESP_LOGI(TAG, "当前为人脸识别模式");
    }
    
    ESP_LOGI(TAG, "当前速度档位: %d", speed);
}
```

### 高级使用示例：手动控制运行模式

```c
#include "voice_module.h"

void manual_mode_control(void)
{
    // 手动切换到命令模式（停止电机）
    voice_module_set_run_mode(0);
    ESP_LOGI(TAG, "手动切换到命令模式");
    
    // 执行一些操作...
    vTaskDelay(pdMS_TO_TICKS(5000));
    
    // 手动切换回巡线模式（启动电机）
    voice_module_set_run_mode(1);
    ESP_LOGI(TAG, "手动切换到巡线模式");
}
```

## 注意事项

### 硬件限制

- ⚠️ GPIO36为仅输入引脚，不支持上拉/下拉电阻
- ⚠️ 语音模块必须与ESP32共地，否则串口通信会出现乱码
- ⚠️ 确保语音模块输出电平为3.3V，如果是5V需要电平转换

### 软件限制

- ⚠️ 命令处理时间应控制在10ms以内，避免影响系统实时性
- ⚠️ 摄像头操作（Camera_SetMode等）可能阻塞较长时间，已优化为不等待结果
- ⚠️ UART接收缓冲区大小为2056字节，如果命令频率过高可能溢出

### 线程安全

- ✅ 所有全局标志变量使用原子类型（`_Atomic`），支持多任务安全访问
- ✅ 建议使用`atomic_load()`和`atomic_store()`函数访问标志变量
- ✅ 语音模块任务固定在Core 1，不影响Core 0的实时控制任务

### 性能考虑

- 语音模块任务优先级为4，低于Timer中断（优先级5），确保实时控制不受影响
- UART接收采用非阻塞模式，超时时间100ms，避免长时间阻塞
- 命令解析采用快速查找算法，处理时间<1ms

## 故障排除

### 常见问题

#### 问题1：接收不到语音命令

**现象**：语音模块发送命令，但ESP32没有响应

**原因**：
1. UART引脚连接错误
2. 波特率不匹配
3. 没有共地

**解决方案**：
1. 检查GPIO36是否正确连接到语音模块TX
2. 确认语音模块波特率设置为115200
3. 使用万用表检查GND连接
4. 使用示波器或逻辑分析仪检查UART信号

#### 问题2：命令解析失败

**现象**：接收到数据但无法解析为有效命令

**原因**：
1. 命令帧格式不正确
2. 串口数据乱码（电平不匹配或干扰）
3. 接收缓冲区溢出

**解决方案**：
1. 检查语音模块发送的命令帧格式是否为`[0xAA] [0x55] [CMD] [0x55] [0xAA]`
2. 使用串口调试工具查看原始数据
3. 增加接收缓冲区大小（修改`VOICE_UART_RX_BUFFER_SIZE`）
4. 降低命令发送频率

#### 问题3：摄像头模式切换失败

**现象**：收到语音命令但摄像头模式没有切换

**原因**：
1. 摄像头通信失败（RS485总线问题）
2. 摄像头设备ID不正确
3. 摄像头未初始化

**解决方案**：
1. 检查摄像头RS485连接
2. 确认摄像头设备ID为3（或修改代码中的`CAMERA_DEVICE_ID`）
3. 确保在调用语音模块前已初始化摄像头
4. 查看摄像头协议组件的日志输出

#### 问题4：运行模式切换无效

**现象**：发送运行模式命令后，电机状态没有变化

**原因**：
1. 电机控制逻辑未检查`Flag_RunMode`标志
2. 标志变量访问方式不正确

**解决方案**：
1. 在电机控制代码中添加运行模式检查：
```c
if (voice_module_get_run_mode() == 0) {
    // 命令模式：停止电机
    pwm_set_duty(0, 0);
}
```
2. 使用`voice_module_get_run_mode()`函数而不是直接访问`Flag_RunMode`

## 命令协议说明

### 命令帧格式

```
[0xAA] [0x55] [CMD] [0x55] [0xAA]
  ↑      ↑      ↑      ↑      ↑
 帧头1  帧头2  命令字  帧尾1  帧尾2
```

### 命令列表

| 命令字 | 功能 | 说明 |
|-------|------|------|
| 0x01 | 颜色识别 | 切换摄像头到颜色识别模式 |
| 0x02 | 色块识别 | 切换摄像头到色块识别模式 |
| 0x03 | 人脸识别 | 切换摄像头到人脸识别模式 |
| 0x04 | 二维码识别 | 切换摄像头到二维码识别模式 |
| 0x05 | 数字识别 | 切换摄像头到数字识别模式 |
| 0x06 | 标签识别 | 切换摄像头到标签识别模式 |
| 0x07 | 20类识别 | 切换摄像头到20类物体识别模式 |
| 0x09 | 拍照 | 摄像头拍照 |
| 0x0B | 复位 | 摄像头复位 |
| 0x0C | 增加亮度 | 摄像头亮度+1（最大4） |
| 0x0D | 降低亮度 | 摄像头亮度-1（最小0） |
| 0x0E | 高速模式 | 速度设置为2（基础速度999） |
| 0x0F | 低速模式 | 速度设置为1（基础速度700） |
| 0x11 | 巡线模式 | 启动电机，开始巡线 |
| 0x12 | 命令模式 | 停止电机，等待命令 |
