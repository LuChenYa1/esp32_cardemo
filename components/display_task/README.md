# Display Task - 数码管轮播显示任务

## 元信息

- **版本**: 1.0.0
- **更新日期**: 2024-12-20
- **维护者**: 项目团队
- **使用状态**: main.c使用中

---

## 概述

数码管轮播显示任务实现了多传感器数据的自动轮播显示功能，每5秒自动切换显示模式，每200ms刷新一次数码管。

## 功能特性

### 显示模式

1. **光照模式**
   - 显示光照传感器ADC原始值（0-4095）
   - 使用ADC1_CH0（GPIO36）读取

2. **温湿度模式**
   - 显示格式：TTHH（前2位温度℃，后2位湿度%）
   - 使用DHT11传感器（GPIO38）读取
   - 读取失败时自动保持上次有效值

3. **距离模式**
   - 显示距离值（单位：cm，范围0-9999）
   - 使用VL53L0X测距传感器（I2C GPIO39/40）读取

### 轮播逻辑

- **刷新间隔**：200ms
- **模式切换间隔**：5秒
- **切换顺序**：光照 → 温湿度 → 距离 → 光照（循环）

## 硬件连接

| 外设 | GPIO | 说明 |
|------|------|------|
| 光照传感器 | GPIO36 (ADC1_CH0) | 模拟输入 |
| DHT11数据 | GPIO38 | 数字输入/输出 |
| VL53L0X SDA | GPIO39 | I2C数据线 |
| VL53L0X SCL | GPIO40 | I2C时钟线 |
| TM1637 DIO | GPIO33 | 数码管数据 |
| TM1637 CLK | GPIO34 | 数码管时钟 |

## API接口

### display_task()

```c
void display_task(void *pvParameters);
```

数码管轮播显示任务入口函数。

**功能**：
- 初始化所有传感器（TM1637、DHT11、VL53L0X、光照传感器）
- 每200ms刷新数码管显示
- 每5秒切换显示模式
- DHT11读取失败时保持上次有效值

**参数**：
- `pvParameters`：任务参数（未使用）

### display_task_create()

```c
esp_err_t display_task_create(void);
```

创建数码管显示任务。

**返回值**：
- `ESP_OK`：成功
- `ESP_FAIL`：失败

## 使用示例

```c
#include "display_task.h"

void app_main(void)
{
    // 创建数码管显示任务
    esp_err_t ret = display_task_create();
    if (ret != ESP_OK) {
        ESP_LOGE("MAIN", "数码管显示任务创建失败");
        return;
    }
    
    ESP_LOGI("MAIN", "数码管显示任务已启动");
}
```

## 配置参数

可在 `display_task.h` 中修改以下参数：

```c
#define DISPLAY_TASK_PRIORITY       3       // 任务优先级
#define DISPLAY_TASK_STACK_SIZE     4096    // 任务栈大小
#define DISPLAY_REFRESH_MS          200     // 刷新间隔（毫秒）
#define DISPLAY_MODE_INTERVAL_MS    5000    // 模式切换间隔（毫秒）
```

## 错误处理

### DHT11读取失败

当DHT11读取失败时（超时或校验和错误），任务会：
1. 记录警告日志
2. 使用上次有效值继续显示
3. 不会中断任务执行

默认有效值：
- 温度：25℃
- 湿度：50%

### 光照传感器读取失败

当ADC读取失败时，返回0值并记录警告日志。

### 距离传感器读取失败

VL53L0X内部有错误处理机制，异常值会被过滤。

## 性能指标

- **任务优先级**：3（低于Timer中断）
- **刷新周期**：200ms
- **模式切换周期**：5秒
- **单次刷新时间**：< 10ms（不包括传感器读取）
- **DHT11读取时间**：约20-30ms
- **VL53L0X读取时间**：约5-10ms
- **光照ADC读取时间**：< 1ms

## 依赖组件

- `driver`：GPIO、ADC驱动
- `esp_adc`：ADC oneshot API
- `tm1637`：TM1637数码管驱动
- `dht11`：DHT11温湿度传感器驱动
- `vl53l0`：VL53L0X测距传感器驱动
- `i2c`：I2C配置

## 注意事项

1. **ADC通道选择**：光照传感器使用ADC1_CH0，避免与WiFi冲突（ADC2与WiFi互斥）
2. **DHT11读取间隔**：DHT11建议读取间隔≥2秒，本任务每5秒读取一次，符合要求
3. **I2C总线共享**：VL53L0X使用独立的I2C引脚（GPIO39/40），不与其他I2C设备冲突
4. **任务优先级**：优先级3低于Timer中断（优先级1-2），确保实时任务不受影响
