# 五路灰度传感器模块（I2C）

## 概述

五路灰度传感器模块通过I2C接口读取5个灰度传感器的数据，用于高精度巡线检测。相比二路ADC灰度传感器，五路传感器提供更精确的黑线位置信息，适合复杂巡线场景。

## 硬件连接

### 引脚分配

- **SCL**: GPIO18（复用二路灰度传感器左传感器引脚）
- **SDA**: GPIO20（复用二路灰度传感器右传感器引脚）
- **VCC**: 3.3V/5V电源
- **GND**: 地线

### I2C配置

- **总线**: I2C_NUM_1（独立I2C总线）
- **地址**: 0x4F（7位地址）
- **速率**: 400 kHz
- **上拉**: 使用ESP32内部上拉电阻

### 注意事项

⚠️ **与二路ADC灰度传感器互斥使用**
- 五路I2C灰度传感器与二路ADC灰度传感器共用GPIO18/20
- 只能初始化其中一个模块，不能同时使用
- 根据实际需求选择使用二路或五路传感器

## 数据格式

### I2C通信协议

传感器通过I2C返回10字节数据：
```
字节0-1: 传感器1值（小端序，低字节在前）
字节2-3: 传感器2值
字节4-5: 传感器3值
字节6-7: 传感器4值
字节8-9: 传感器5值
```

### 数据范围

- **原始值**: 0-1023（10位ADC）
- **黑线**: 值较小（< 512）
- **白色**: 值较大（> 512）

**注意**: 传感器实际输出为10位ADC值（0-1023），不是12位（0-4095）。这是传感器硬件的特性。

## 工作原理

### 后台扫描任务

模块初始化时会创建一个后台FreeRTOS任务，自动定期读取传感器数据：

- **任务名称**: `five_gray_scan`
- **优先级**: 6（与二路灰度扫描任务相同）
- **栈大小**: 2048字节
- **扫描周期**: 2ms（500 Hz采样率）

### 数据缓存机制

- 使用原子变量（`_Atomic`）保护共享数据
- 后台任务自动更新缓存
- 用户代码通过API读取缓存数据（中断安全）
- 读取失败时保持上次有效值

### 错误处理

- 连续读取失败时记录错误计数
- 连续失败100次进入安全模式（调用`enter_safe_mode()`）
- 提供统计信息（读取次数、错误次数、成功率）

## 使用示例

### 基本使用

```c
#include "five_way_gray_i2c.h"

void app_main(void)
{
    // 初始化传感器（启动后台扫描任务）
    esp_err_t ret = five_way_gray_i2c_init();
    if (ret != ESP_OK) {
        ESP_LOGE(TAG, "五路灰度传感器初始化失败");
        return;
    }
    
    while (1) {
        // 读取所有传感器值
        uint16_t values[5];
        five_way_gray_i2c_get_values(values);
        
        printf("传感器值: [%d, %d, %d, %d, %d]\n",
               values[0], values[1], values[2], values[3], values[4]);
        
        vTaskDelay(pdMS_TO_TICKS(100));
    }
}
```

### 黑线检测

```c
// 检测是否有任意传感器检测到黑线
if (five_way_gray_i2c_any_black_detected(0)) {
    printf("检测到黑线！\n");
}

// 使用自定义阈值
if (five_way_gray_i2c_any_black_detected(800)) {
    printf("检测到黑线（阈值=800）！\n");
}
```

### 读取单个传感器

```c
// 读取中间传感器（索引2）
uint16_t center_value = five_way_gray_i2c_get_value(2);
printf("中间传感器值: %d\n", center_value);
```

### 获取统计信息

```c
uint32_t read_count, error_count;
five_way_gray_i2c_get_stats(&read_count, &error_count);

float success_rate = (read_count > 0) ? 
    (100.0 * (read_count - error_count) / read_count) : 0.0;

printf("读取次数: %lu, 错误次数: %lu, 成功率: %.2f%%\n",
       read_count, error_count, success_rate);
```

### 调试输出

```c
// 打印完整的传感器状态
five_way_gray_i2c_print_status();
```

输出示例：
```
========================================
五路灰度传感器状态
========================================
传感器值: [3200, 3150, 650, 3180, 3210]
黑线检测: [白, 白, 黑, 白, 白] (阈值=700)
统计信息: 读取次数=5000, 错误次数=2, 成功率=99.96%
========================================
```

## API参考

### 初始化函数

#### `five_way_gray_i2c_init()`

初始化五路灰度传感器，配置I2C总线并启动后台扫描任务。

**返回值:**
- `ESP_OK`: 初始化成功
- `ESP_FAIL`: 初始化失败

**注意:**
- 必须在使用其他API前调用
- 会注册GPIO到gpio_manager，检测引脚冲突
- 自动创建后台扫描任务

### 数据读取函数

#### `five_way_gray_i2c_get_values(uint16_t values[5])`

获取所有传感器的缓存值（中断安全）。

**参数:**
- `values`: 5个uint16_t的数组，用于存储传感器值

**特性:**
- 使用原子操作，可在中断中安全调用
- 读取后台任务缓存的最新数据
- 不会阻塞，立即返回

#### `five_way_gray_i2c_get_value(uint8_t index)`

获取单个传感器的值。

**参数:**
- `index`: 传感器索引（0-4）

**返回值:**
- 传感器值（0-4095）
- 索引无效时返回0

### 黑线检测函数

#### `five_way_gray_i2c_any_black_detected(uint16_t threshold)`

检测是否有任意传感器检测到黑线。

**参数:**
- `threshold`: 黑线检测阈值，传感器值小于此值判定为黑线
  - 使用0则采用默认阈值（700）

**返回值:**
- `true`: 至少一个传感器检测到黑线
- `false`: 所有传感器都在白色区域

### 统计函数

#### `five_way_gray_i2c_get_stats(uint32_t *read_count, uint32_t *error_count)`

获取统计信息。

**参数:**
- `read_count`: 输出参数，累计读取次数，可为NULL
- `error_count`: 输出参数，累计错误次数，可为NULL

### 调试函数

#### `five_way_gray_i2c_print_status()`

打印传感器状态，包括：
- 5个传感器的当前值
- 黑线检测状态
- 统计信息（读取次数、错误次数、成功率）

## 故障排除

### 初始化失败

**问题**: `five_way_gray_i2c_init()` 返回失败

**可能原因:**
1. GPIO18/20被其他模块占用（如二路ADC灰度传感器）
2. I2C总线初始化失败
3. 传感器未正确连接

**解决方法:**
- 检查是否同时初始化了二路ADC灰度传感器
- 确认硬件连接正确
- 查看日志输出的详细错误信息

### 数据读取失败

**问题**: 错误计数持续增加

**可能原因:**
1. I2C地址不正确（默认0x4F）
2. 传感器未上电或损坏
3. I2C总线干扰

**解决方法:**
- 确认传感器I2C地址
- 检查电源和地线连接
- 缩短I2C总线长度，减少干扰
- 考虑添加外部上拉电阻（4.7kΩ）

### 数据未就绪

**问题**: `five_way_gray_i2c_print_status()` 显示"数据未就绪"

**可能原因:**
- 后台任务尚未成功读取数据
- I2C通信失败

**解决方法:**
- 等待几毫秒后重试
- 检查I2C通信是否正常

## 配置选项

### 修改I2C速率

在 `pin_definitions.h` 中修改：
```c
#define FIVE_WAY_GRAY_I2C_FREQ_HZ   400000  // 400kHz（推荐）
// 可选：100000 (100kHz), 1000000 (1MHz)
```

### 修改扫描周期

在 `five_way_gray_i2c.c` 中修改：
```c
vTaskDelay(pdMS_TO_TICKS(2));  // 2ms周期
```

### 修改黑线阈值

在 `five_way_gray_i2c.h` 中修改：
```c
#define FIVE_WAY_GRAY_BLACK_THRESHOLD  512  // 默认阈值（10位ADC，范围0-1023）
```

或在代码中使用自定义阈值：
```c
five_way_gray_i2c_any_black_detected(600);  // 使用600作为阈值
```

**阈值建议：**
- 白色表面：800-1000
- 黑线：100-400
- 推荐阈值：512（中间值）
- 可根据实际环境调整
