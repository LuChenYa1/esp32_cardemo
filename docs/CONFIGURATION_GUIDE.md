# 配置参数指南

## 文档说明

本文档详细说明ESP32巡线小车项目中所有可调参数的含义、取值范围、默认值和调整方法。通过调整这些参数，可以优化系统性能、适应不同环境和硬件配置。

**更新日期**: 2024-12-20  
**项目**: ESP32巡线小车  
**文档版本**: 1.0

### 文档用途

- 系统调试：调整参数优化性能
- 环境适配：根据不同场地调整传感器参数
- 性能优化：平衡速度和稳定性
- 故障排查：通过参数调整解决问题

---

## 参数分类

本项目的配置参数分为以下几类：

1. **PD控制参数**：影响巡线控制的响应速度和稳定性
2. **传感器校准参数**：灰度传感器的黑白阈值
3. **定时器周期参数**：实时任务的执行频率
4. **电机PWM参数**：电机速度和转向控制
5. **串口通信参数**：UART波特率和缓冲区大小
6. **转弯检测参数**：转弯判定阈值和延迟
7. **显示更新参数**：数码管显示周期

---

## PD控制参数

### 参数概述

PD控制器是巡线小车的核心算法，通过比例-微分控制实现平滑的路径跟踪。参数调整直接影响小车的响应速度、稳定性和巡线精度。

### 参数列表

| 参数名称 | 默认值 | 取值范围 | 单位 | 代码位置 |
|---------|--------|---------|------|---------|
| DEFAULT_SPEED | 700 | 0-1023 | PWM占空比 | `pd_controller.c` |
| DEFAULT_KP | 6.0 | 0.0-100.0 | 无量纲 | `pd_controller.c` |
| DEFAULT_KD | 30.0 | 0.0-100.0 | 无量纲 | `pd_controller.c` |
| ERROR_DEADZONE | 50.0 | 0.0-510.0 | 无量纲 | `pd_controller.c` |

### 参数详细说明

#### 1. DEFAULT_SPEED（基础速度）

**含义**：小车直线行驶时的基础速度，对应电机PWM占空比。

**取值范围**：
- 最小值：0（停止）
- 最大值：1023（全速）
- 推荐范围：400-800

**默认值**：700

**影响**：
- 值越大，小车速度越快，但转弯可能不够灵活
- 值越小，小车速度越慢，但转弯更加精确
- 过大可能导致传感器响应不及时，偏离轨道
- 过小可能导致电机扭矩不足，无法启动

**调整建议**：
1. 初次调试从400开始，逐步增加
2. 在直线上测试，确保能够稳定跟踪
3. 在弯道上测试，确保能够顺利转弯
4. 根据场地情况调整（光滑地面可以更快）

**修改方法**：
```c
// 方法1：修改源代码（需要重新编译）
// 文件：components/pd_controller/pd_controller.c
#define DEFAULT_SPEED 700  // 修改此值

// 方法2：运行时动态调整（推荐）
pd_controller_set_params(800, 6.0f, 30.0f);  // 设置速度为800
```

#### 2. DEFAULT_KP（比例系数）

**含义**：PD控制器的比例项系数，控制对当前误差的响应强度。

**取值范围**：
- 最小值：0.0（无比例控制）
- 最大值：100.0（理论上限）
- 推荐范围：3.0-10.0

**默认值**：6.0

**影响**：
- Kp越大，对误差的响应越快，纠偏速度越快
- Kp过大会导致震荡，小车左右摆动
- Kp过小会导致响应迟钝，无法及时纠偏

**调整建议**：
1. 从3.0开始，逐步增加0.5
2. 观察小车是否能够快速回到轨道中心
3. 如果出现左右摆动，减小Kp
4. 如果纠偏太慢，增大Kp
5. 速度越快，需要的Kp越大

**修改方法**：
```c
// 方法1：修改源代码
#define DEFAULT_KP 6.0f  // 修改此值

// 方法2：运行时动态调整（推荐）
pd_controller_set_params(700, 8.0f, 30.0f);  // 设置Kp为8.0
```

#### 3. DEFAULT_KD（微分系数）

**含义**：PD控制器的微分项系数，控制对误差变化率的响应强度，起到阻尼作用。

**取值范围**：
- 最小值：0.0（无微分控制）
- 最大值：100.0（理论上限）
- 推荐范围：10.0-50.0

**默认值**：30.0

**影响**：
- Kd越大，阻尼越强，越稳定，但响应可能变慢
- Kd过大会导致反应迟钝，无法快速纠偏
- Kd过小会导致震荡，小车左右摆动

**调整建议**：
1. 先调整Kp，再调整Kd
2. 如果Kp调整后仍有震荡，增大Kd
3. 如果响应太慢，减小Kd
4. Kd通常是Kp的3-6倍

**修改方法**：
```c
// 方法1：修改源代码
#define DEFAULT_KD 30.0f  // 修改此值

// 方法2：运行时动态调整（推荐）
pd_controller_set_params(700, 6.0f, 40.0f);  // 设置Kd为40.0
```

#### 4. ERROR_DEADZONE（误差死区）

**含义**：当误差小于此值时，认为小车在轨道中心，不进行调整。

**取值范围**：
- 最小值：0.0（无死区）
- 最大值：510.0（最大误差）
- 推荐范围：30.0-100.0

**默认值**：50.0

**影响**：
- 死区越大，小车越稳定，但精度降低
- 死区越小，精度越高，但可能出现抖动
- 死区为0时，小车会持续微调，可能抖动

**调整建议**：
1. 如果小车在直线上抖动，增大死区
2. 如果小车偏离轨道中心，减小死区
3. 根据传感器精度调整（传感器噪声大时增大死区）

**修改方法**：
```c
// 修改源代码
#define ERROR_DEADZONE 50.0f  // 修改此值
```

### PD参数调整流程

#### 步骤1：低速测试

```c
// 设置低速，使用默认参数
pd_controller_set_params(400, 6.0f, 30.0f);
pd_controller_enable();
```

观察小车行为：
- ✅ 能够跟踪轨道 → 进入步骤2
- ❌ 左右摆动 → 减小Kp或增大Kd
- ❌ 反应迟钝 → 增大Kp

#### 步骤2：调整Kp

```c
// 逐步增加Kp，观察响应速度
pd_controller_set_params(400, 7.0f, 30.0f);  // Kp = 7.0
vTaskDelay(pdMS_TO_TICKS(5000));

pd_controller_set_params(400, 8.0f, 30.0f);  // Kp = 8.0
vTaskDelay(pdMS_TO_TICKS(5000));
```

找到最佳Kp：
- 响应快速但不震荡
- 能够快速回到轨道中心

#### 步骤3：调整Kd

```c
// 在最佳Kp基础上调整Kd
pd_controller_set_params(400, 8.0f, 35.0f);  // Kd = 35.0
vTaskDelay(pdMS_TO_TICKS(5000));

pd_controller_set_params(400, 8.0f, 40.0f);  // Kd = 40.0
vTaskDelay(pdMS_TO_TICKS(5000));
```

找到最佳Kd：
- 稳定不震荡
- 响应不太慢

#### 步骤4：提高速度

```c
// 逐步提高速度，重新调整参数
pd_controller_set_params(500, 8.0f, 40.0f);
vTaskDelay(pdMS_TO_TICKS(5000));

pd_controller_set_params(600, 8.0f, 40.0f);
vTaskDelay(pdMS_TO_TICKS(5000));

pd_controller_set_params(700, 8.0f, 40.0f);
vTaskDelay(pdMS_TO_TICKS(5000));
```

注意：速度越快，可能需要更大的Kp和Kd

### PD参数推荐值

| 速度范围 | Kp推荐 | Kd推荐 | 适用场景 |
|---------|--------|--------|---------|
| 300-500 | 4.0-6.0 | 20.0-30.0 | 低速，稳定优先，适合复杂轨道 |
| 500-700 | 6.0-8.0 | 30.0-40.0 | 中速，平衡性能，适合一般轨道 |
| 700-900 | 8.0-10.0 | 40.0-50.0 | 高速，响应优先，适合简单轨道 |

## 传感器校准参数

### 灰度传感器校准

灰度传感器用于检测地面的黑白线条，校准参数直接影响巡线精度。

### 参数列表

| 参数名称 | 默认值 | 取值范围 | 单位 | 代码位置 |
|---------|--------|---------|------|---------|
| LEFT_WHITE_VALUE | 4095 | 2000-4095 | ADC值 | `pd_controller.c` |
| LEFT_BLACK_VALUE | 1269 | 0-2000 | ADC值 | `pd_controller.c` |
| RIGHT_WHITE_VALUE | 4095 | 2000-4095 | ADC值 | `pd_controller.c` |
| RIGHT_BLACK_VALUE | 1354 | 0-2000 | ADC值 | `pd_controller.c` |

### 参数详细说明

#### 1. LEFT_WHITE_VALUE / RIGHT_WHITE_VALUE（白色区域ADC值）

**含义**：传感器在白色区域（地面）上的ADC读数。

**取值范围**：
- 最小值：2000（较暗的白色）
- 最大值：4095（纯白色）
- 典型值：3400-4095

**默认值**：4095

**影响**：
- 用于归一化计算，将ADC值映射到0.0-1.0范围
- 值越大，表示白色区域越亮
- 不准确会导致归一化值错误，影响巡线精度

**校准方法**：
```c
// 方法1：自动校准（推荐）
gray_sensor_calibrate();  // 按照提示将传感器放在白色区域

// 方法2：手动设置
gray_sensor_set_calibration(GRAY_SENSOR_LEFT, 4095, 1269);
gray_sensor_set_calibration(GRAY_SENSOR_RIGHT, 4095, 1354);
```

#### 2. LEFT_BLACK_VALUE / RIGHT_BLACK_VALUE（黑线区域ADC值）

**含义**：传感器在黑线上的ADC读数。

**取值范围**：
- 最小值：0（纯黑色）
- 最大值：2000（较浅的黑色）
- 典型值：800-1500

**默认值**：左1269，右1354

**影响**：
- 用于归一化计算
- 值越小，表示黑线越黑
- 不准确会导致无法正确识别黑线

**校准方法**：
```c
// 自动校准（推荐）
gray_sensor_calibrate();  // 按照提示将传感器放在黑线上
```

### 校准步骤

#### 步骤1：准备工作

1. 确保传感器已正确连接
2. 确保传感器距离地面5-10mm
3. 确保光照条件稳定

#### 步骤2：执行自动校准

```c
#include "gray_sensor.h"

void calibrate_sensors(void) {
    ESP_LOGI(TAG, "开始灰度传感器校准...");
    
    // 调用自动校准函数
    gray_sensor_calibrate();
    
    ESP_LOGI(TAG, "校准完成");
}
```

校准过程：
1. 程序提示"请将传感器放在白色区域，按Enter继续"
2. 将小车放在白色地面上，按Enter
3. 程序采集白色区域ADC值
4. 程序提示"请将传感器放在黑线上，按Enter继续"
5. 将小车放在黑线上，按Enter
6. 程序采集黑线ADC值
7. 校准参数自动保存

#### 步骤3：验证校准结果

```c
// 打印传感器状态
gray_sensor_print_status();
```

输出示例：
```
左传感器: 原始值=3400, 归一化=0.95, 黑线检测=否
右传感器: 原始值=3380, 归一化=0.94, 黑线检测=否
```

#### 步骤4：测试巡线效果

```c
// 启用PD控制器，测试巡线
pd_controller_enable();
```

观察小车行为：
- ✅ 能够稳定跟踪黑线 → 校准成功
- ❌ 偏离黑线或震荡 → 重新校准或调整PD参数

### 校准注意事项

⚠️ **环境光影响**：
- 不同光照条件下，ADC值会有差异
- 建议在实际使用环境中校准
- 如果环境光变化大，需要重新校准

⚠️ **传感器高度**：
- 传感器距离地面5-10mm最佳
- 过高会导致灵敏度降低
- 过低可能碰到地面

⚠️ **地面材质**：
- 不同材质的反射率不同
- 建议在实际轨道上校准
- 黑线宽度应与传感器间距匹配

## 定时器周期参数

### 参数概述

定时器系统控制实时任务的执行频率，影响系统响应速度和CPU负载。

### 参数列表

| 参数名称 | 默认值 | 取值范围 | 单位 | 代码位置 |
|---------|--------|---------|------|---------|
| TIMER_0_INTERVAL_MS | 3 | 1-10 | 毫秒 | `timer_system.c` |
| TIMER_1_INTERVAL_MS | 10 | 5-50 | 毫秒 | `timer_system.c` |

### 参数详细说明

#### 1. TIMER_0_INTERVAL_MS（Timer 0周期）

**含义**：Timer 0中断周期，用于灰度扫描和转弯检测。

**取值范围**：
- 最小值：1ms（高频，CPU负载大）
- 最大值：10ms（低频，响应慢）
- 推荐值：3ms

**默认值**：3ms

**影响**：
- 周期越短，转弯检测越及时，但CPU负载越大
- 周期越长，CPU负载越小，但可能错过转弯
- 每秒触发次数 = 1000 / 周期

**调整建议**：
- 一般情况使用默认值3ms
- 高速巡线可以减小到2ms
- 低速巡线可以增大到5ms

**修改方法**：
```c
// 修改源代码（需要重新编译）
// 文件：components/timer_system/timer_system.c
#define TIMER_0_INTERVAL_MS 3  // 修改此值
```

#### 2. TIMER_1_INTERVAL_MS（Timer 1周期）

**含义**：Timer 1中断周期，用于PD控制和转弯状态机。

**取值范围**：
- 最小值：5ms（高频，控制精细）
- 最大值：50ms（低频，控制粗糙）
- 推荐值：10ms

**默认值**：10ms

**影响**：
- 周期越短，控制越精细，但CPU负载越大
- 周期越长，CPU负载越小，但控制精度降低
- 每秒触发次数 = 1000 / 周期

**调整建议**：
- 一般情况使用默认值10ms
- 高速巡线可以减小到8ms
- 低速巡线可以增大到15ms

**修改方法**：
```c
// 修改源代码
#define TIMER_1_INTERVAL_MS 10  // 修改此值
```

### 定时器周期调整建议

| 应用场景 | Timer 0周期 | Timer 1周期 | 说明 |
|---------|------------|------------|------|
| 标准巡线 | 3ms | 10ms | 默认配置，平衡性能 |
| 高速巡线 | 2ms | 8ms | 提高响应速度 |
| 低速巡线 | 5ms | 15ms | 降低CPU负载 |
| 调试模式 | 10ms | 20ms | 降低中断频率，便于调试 |

## 电机PWM参数

### 参数概述

电机PWM参数控制电机的速度和转向，影响小车的运动性能。

### 参数列表

| 参数名称 | 默认值 | 取值范围 | 单位 | 代码位置 |
|---------|--------|---------|------|---------|
| PWM_FREQUENCY | 1000 | 100-20000 | Hz | `pwm.c` |
| PWM_RESOLUTION | 10 | 8-14 | 位 | `pwm.c` |
| PWM_MAX_DUTY | 1023 | 0-16383 | 无量纲 | `pwm.c` |

### 参数详细说明

#### 1. PWM_FREQUENCY（PWM频率）

**含义**：电机PWM信号的频率。

**取值范围**：
- 最小值：100Hz（低频，可能有噪音）
- 最大值：20000Hz（高频，超声波范围）
- 推荐值：1000Hz-5000Hz

**默认值**：1000Hz

**影响**：
- 频率越高，电机运转越平滑，噪音越小
- 频率过高可能导致电机驱动板响应不及时
- 频率过低可能产生可闻噪音

**修改方法**：
```c
// 修改源代码
#define PWM_FREQUENCY 1000  // 修改此值
```

#### 2. PWM_RESOLUTION（PWM分辨率）

**含义**：PWM占空比的分辨率（位数）。

**取值范围**：
- 最小值：8位（256级）
- 最大值：14位（16384级）
- 推荐值：10位（1024级）

**默认值**：10位

**影响**：
- 分辨率越高，速度控制越精细
- 分辨率越高，PWM频率上限越低
- 10位分辨率提供0-1023的占空比范围

**修改方法**：
```c
// 修改源代码
#define PWM_RESOLUTION 10  // 修改此值
```
