# 程序设计思路

## 一、整体设计思路

本项目采用分层模块化架构设计，将硬件驱动、功能模块和应用逻辑分离。使用ESP32硬件定时器实现高频实时任务，FreeRTOS任务处理低频外设操作，通过原子变量保护共享数据，实现多任务协同工作。

### 1. 系统架构（四层结构）

**第一层：应用层（main.c）**
- 主循环实现语音控制和系统监控
- 协调各模块工作，实现整体功能

**第二层：任务调度层**
- 硬件定时器：Timer 0（3ms周期）、Timer 1（10ms周期）
- FreeRTOS任务：ADC采样、显示、舵机、语音、交通灯

**第三层：功能模块层**
- 转弯检测模块、PD控制器、转弯状态机
- 实现循迹和转弯控制逻辑

**第四层：硬件抽象层（components）**
- 传感器驱动、执行器驱动、通信接口、板级配置

### 2. 初始化流程（7个步骤）

1. GPIO管理器初始化：统一管理引脚分配，防止冲突
2. 硬件外设初始化：PWM、UART、ADC、传感器等
3. 软件模块初始化：转弯检测、PD控制器、状态机
4. 系统自检：验证关键模块正常工作
5. 定时器系统初始化：配置硬件定时器中断
6. 创建FreeRTOS任务：ADC采样、显示、舵机、语音等
7. 启动系统：启动定时器，进入主循环

### 3. 主循环逻辑（200ms周期）

主循环每200ms执行一次，主要功能：
- 检查语音模块命令，控制电机启停（巡线模式/命令模式）
- 系统状态监控，定期打印运行状态
- 看门狗喂狗，防止系统死锁

## 二、分模块说明（按运行流程）

### 1. ADC采样任务（优先级6，1ms周期）

**功能**：高频采样灰度传感器ADC值

**运行流程**：
1. 每1ms触发一次采样
2. 读取GPIO18（左灰度）和GPIO20（右灰度）的ADC值
3. 将数据缓存到原子变量中
4. 供Timer 0和其他任务读取

**作用**：为循迹控制提供实时的灰度传感器数据

---

### 2. Timer 0任务（3ms周期）

**功能**：灰度传感器扫描和转弯检测

**运行流程**：
1. 从原子变量读取灰度传感器缓存值
2. 调用转弯检测模块分析数据
3. 判断是否在黑线上、是否检测到转弯
4. 更新转弯类型（左转、右转、十字路口、T字路口）

**作用**：实时检测路径状态，为控制决策提供依据

---

### 3. Timer 1任务（10ms周期）

**功能**：PD控制和转弯状态机

**运行流程**：
1. 读取灰度传感器值，计算偏差
2. PD控制器根据偏差计算左右电机速度
3. 转弯状态机根据转弯检测结果切换状态
4. 控制电机执行相应动作（直行、转弯、停止）

**作用**：实现循迹控制和转弯动作执行

---

### 4. 语音模块任务（优先级4）

**功能**：接收语音命令，控制系统运行模式

**运行流程**：
1. 通过UART1接收语音模块发送的命令
2. 解析命令类型（巡线模式、命令模式、摄像头识别等）
3. 更新运行模式标志（原子变量）
4. 根据命令控制摄像头进行识别（颜色、人脸、二维码等）

**作用**：实现语音控制，切换系统工作模式

---

### 5. 数码管显示任务（优先级3，5秒轮播）

**功能**：轮流显示传感器数据

**运行流程**：
1. 每5秒切换显示内容
2. 依次显示：温湿度（DHT11）→ 超声波测距（HC-SR04）→ 红外避障状态
3. 调用TM1637驱动更新数码管显示

**作用**：实时显示传感器状态，便于调试和监控

---

### 6. 舵机控制任务（优先级3）

**功能**：控制485舵机执行动作

**运行流程**：
1. 接收舵机控制命令（位置、速度）
2. 通过UART0（RS485）发送舵机控制指令
3. 控制RS485方向引脚（GPIO19）切换收发模式
4. 等待舵机响应

**作用**：控制机械臂、云台等舵机执行动作

---

### 7. 交通灯控制任务（优先级2）

**功能**：模拟交通信号灯

**运行流程**：
1. 每3秒切换一次灯光状态
2. 按照红灯→绿灯→黄灯顺序循环
3. 通过GPIO39/40的二进制编码控制灯光

**作用**：演示交通灯控制，可用于路口识别训练

---

### 8. 主循环（应用层）

**功能**：语音控制和系统监控

**运行流程**：
1. 每200ms执行一次
2. 检查语音模块的运行模式标志
3. 根据模式启用/禁用PD控制器：
   - 巡线模式：启用PD控制器，电机自动循迹
   - 命令模式：禁用PD控制器，停止电机
4. 定期打印系统状态（定时器计数、ADC错误率、灰度值等）

**作用**：协调各模块工作，实现整体功能

## 三、数据流说明

### 1. 灰度传感器数据流
灰度传感器 → ADC采样任务 → 原子变量缓存 → Timer 0读取 → 转弯检测模块 → Timer 1读取 → PD控制器 → 电机PWM输出

### 2. 语音控制数据流
语音模块 → UART1接收 → 语音任务解析 → 运行模式标志（原子变量）→ 主循环读取 → 启用/禁用PD控制器

### 3. 摄像头识别数据流
摄像头 → RS485通信 → 摄像头协议解析 → 识别结果标志（原子变量）→ 语音任务读取

### 4. 转弯控制数据流
转弯检测模块 → 转弯类型判断 → 转弯状态机 → 状态切换 → 电机控制 → 执行转弯动作

## 四、关键设计要点

### 1. 实时性保证
- 高频任务（灰度扫描、转弯检测）使用硬件定时器中断，优先级最高
- 控制任务（PD控制、状态机）使用10ms定时器，满足控制周期要求
- 低频任务（显示、语音）使用FreeRTOS任务，不影响实时性

### 2. 数据保护
- 所有共享变量使用原子操作（_Atomic）
- 中断和任务之间通过原子变量安全通信
- 避免使用互斥锁，防止优先级反转

### 3. 模块解耦
- 每个组件独立封装，提供统一接口
- 上层模块通过接口调用，不直接访问硬件
- 便于移植和维护

### 4. 错误处理
- 初始化失败立即停止系统
- ADC读取失败计数，超过阈值进入安全模式
- 看门狗定时器防止系统死锁
