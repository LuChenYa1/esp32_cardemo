- 代码块格式正确，可以直接使用
- 代码块缩进一致

### 3.2 代码块格式示例

**C语言代码块：**

````markdown
```c
/**
 * @brief 初始化灰度传感器
 * 
 * @return esp_err_t 初始化结果
 */
esp_err_t gray_sensor_init(void) {
    // 配置GPIO为输入模式
    gpio_config_t io_conf = {
        .pin_bit_mask = GRAY_SENSOR_PIN_MASK,
        .mode = GPIO_MODE_INPUT,
        .pull_up_en = GPIO_PULLUP_DISABLE,
        .pull_down_en = GPIO_PULLDOWN_DISABLE,
        .intr_type = GPIO_INTR_DISABLE
    };
    
    return gpio_config(&io_conf);
}
```
````

````markdown

```
````


## 格式规范总结

### Markdown格式最佳实践

1. **标题层级**
   - 一级标题只用于文档标题
   - 二级标题用于主要章节
   - 三级标题用于子章节
   - 不跳级使用标题

2. **表格格式**
   - 始终包含表头
   - 使用对齐符号
   - 保持列宽适中
   - 内容完整无空单元格

3. **代码块格式**
   - 指定语言类型
   - 添加中文注释
   - 保持缩进一致
   - 确保代码可运行

4. **列表格式**
   - 有序列表用于步骤
   - 无序列表用于要点
   - 保持缩进一致
   - 嵌套层级清晰

5. **元信息**
   - 每个文档包含元信息
   - 格式统一
   - 信息完整
   - 及时更新

6. **标点符号**
   - 中文内容用中文标点
   - 英文内容用英文标点
   - 保持一致性

7. **链接格式**
   - 使用相对路径
   - 描述文本清晰
   - 格式统一
