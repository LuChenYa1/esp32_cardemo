## 注释规范总结

### 代码注释最佳实践

1. **文件头注释**
   - 每个源文件都包含文件头注释
   - 说明文件功能和用途
   - 可选包含作者和日期信息

2. **函数注释**
   - 所有公共函数都有注释
   - 说明函数功能、参数、返回值
   - 包含注意事项和警告信息

3. **行内注释**
   - 复杂算法有详细注释
   - 硬件相关参数有说明
   - 魔法数字有含义解释
   - 线程安全相关有标注

4. **注释语言**
   - 使用中文编写
   - 技术术语保留英文
   - 简洁清晰，易于理解

5. **注释维护**
   - 与代码保持同步
   - 及时删除过时注释
   - 标注待办事项和已知问题
