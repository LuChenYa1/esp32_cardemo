# 硬件连接局部原理图

## 文档说明

本文档详细说明了 ESP32-S3 多合一巡线小车的硬件连接关系，包括各个传感器、模块与 ESP32-S3 核心板的引脚对应关系和接口说明。

---

## 1. 系统架构概览

```
┌─────────────────────────────────────────────────────────────┐
│                      ESP32-S3 核心板                         │
│                                                              │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐   │
│  │  电机控制 │  │  编码器   │  │  传感器   │  │  通信接口 │   │
│  │  8个GPIO │  │  8个GPIO │  │  多个GPIO │  │  UART/I2C│   │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘   │
└─────────────────────────────────────────────────────────────┘
         │              │              │              │
         ▼              ▼              ▼              ▼
    电机驱动        编码器模块      各类传感器      外部模块
```

## 2. 传感器接口分配表

### 2.1 SSA 系列接口（Sensor Socket A）

| 接口标识 | GPIO引脚 | 连接模块 | 功能说明 | 使用状态 |
|---------|---------|---------|---------|---------|
| SSA1 | GPIO38 | 红外避障传感器 | 数字输入，检测前方障碍物 | ✅ 使用中 |
| SSA2 | GPIO37 | TM1637数据线 | 数码管显示数据线（DIO） | ⏸️ 未使用 |
| SSA3 | GPIO34 | TM1637时钟线 | 数码管显示时钟线（CLK） | ⏸️ 未使用 |
| SSA4 | GPIO47 | HC-SR04回响 | 超声波测距回响引脚（ECHO） | ⏸️ 未使用 |
| SSA5 | PCF8574-P6 | 扩展IO | 通过I2C扩展芯片提供 | ⏸️ 未使用 |
| SSA6 | GPIO40 | 交通灯信号2 | 交通灯控制信号线2 | ✅ 使用中 |

### 2.2 SSD 系列接口（Sensor Socket D）

| 接口标识 | GPIO引脚 | 连接模块 | 功能说明 | 使用状态 |
|---------|---------|---------|---------|---------|
| SSD3 | GPIO33 | DHT11温湿度 | 单总线数字信号 | ⏸️ 未使用 |
| SSD4 | GPIO48 | HC-SR04触发 | 超声波测距触发引脚（TRIG） | ⏸️ 未使用 |
| SSD5 | PCF8574-P5 | 扩展IO | 通过I2C扩展芯片提供 | ⏸️ 未使用 |
| SSD6 | GPIO39 | 交通灯信号1 | 交通灯控制信号线1 | ✅ 使用中 |

## 3. 详细连接原理图

### 3.1 DHT11 温湿度传感器

```
DHT11 模块                    ESP32-S3 核心板
┌──────────┐                 ┌──────────────┐
│          │                 │              │
│   VCC    │────────────────▶│  3.3V/5V     │
│          │                 │              │
│   DATA   │◀───────────────▶│  GPIO33      │ (SSD3)
│          │    (单总线)      │              │
│   GND    │────────────────▶│  GND         │
│          │                 │              │
└──────────┘                 └──────────────┘
     │
     │  上拉电阻
     │  4.7kΩ-10kΩ
     │
    VCC

说明：
- GPIO33 对应扩展板接口 SSD3
- DHT11 使用单总线协议，需要外接上拉电阻
- 数据线支持双向通信（开漏输出模式）
- 工作电压：3.3V 或 5V
- 测量范围：温度 0-50°C，湿度 20-90%RH
```

### 3.2 HC-SR04 超声波测距传感器

```
HC-SR04 模块                 ESP32-S3 核心板
┌──────────┐                 ┌──────────────┐
│          │                 │              │
│   VCC    │────────────────▶│  5V          │
│          │                 │              │
│   TRIG   │◀────────────────│  GPIO48      │ (SSD4)
│          │    (触发信号)    │              │
│   ECHO   │─────────────────▶│  GPIO47      │ (SSA4)
│          │    (回响信号)    │              │
│   GND    │────────────────▶│  GND         │
│          │                 │              │
└──────────┘                 └──────────────┘

说明：
- GPIO48 (SSD4) 输出触发信号（10us高电平脉冲）
- GPIO47 (SSA4) 接收回响信号（高电平持续时间与距离成正比）
- 测量范围：2cm - 400cm
- 工作电压：5V（ECHO输出5V逻辑，ESP32可承受）
- 测量角度：15度锥形范围
```

### 3.3 灰度传感器（二路ADC）

```
灰度传感器模块               ESP32-S3 核心板
┌──────────┐                 ┌──────────────┐
│          │                 │              │
│   VCC    │────────────────▶│  3.3V        │
│          │                 │              │
│  LEFT    │─────────────────▶│  GPIO18      │ (ADC2_CH7)
│          │   (模拟信号)     │              │
│  RIGHT   │─────────────────▶│  GPIO20      │ (ADC2_CH9)
│          │   (模拟信号)     │              │
│   GND    │────────────────▶│  GND         │
│          │                 │              │
└──────────┘                 └──────────────┘

说明：
- GPIO18 连接左侧灰度传感器（ADC2通道7）
- GPIO20 连接右侧灰度传感器（ADC2通道9）
- 输出电压范围：0-3.3V（黑线=低电压，白色=高电压）
- ⚠️ ADC2 与 WiFi 冲突，启用WiFi时ADC2不可用
- 推荐改用 GPIO47/48（ADC1通道）以兼容WiFi
```

### 3.4 五路灰度传感器（I2C）

```
五路灰度传感器               ESP32-S3 核心板
┌──────────┐                 ┌──────────────┐
│          │                 │              │
│   VCC    │────────────────▶│  3.3V/5V     │
│          │                 │              │
│   SCL    │◀────────────────│  GPIO18      │ (I2C1_SCL)
│          │   (I2C时钟)     │              │
│   SDA    │◀───────────────▶│  GPIO20      │ (I2C1_SDA)
│          │   (I2C数据)     │              │
│   GND    │────────────────▶│  GND         │
│          │                 │              │
└──────────┘                 └──────────────┘

说明：
- 使用独立I2C总线（I2C_NUM_1）
- I2C地址：0x4F
- I2C频率：400kHz
- 与二路ADC灰度传感器复用GPIO18/20，不能同时使用
- 后台任务2ms周期自动更新数据
```

### 3.5 红外避障传感器

```
红外避障传感器               ESP32-S3 核心板
┌──────────┐                 ┌──────────────┐
│          │                 │              │
│   VCC    │────────────────▶│  3.3V/5V     │
│          │                 │              │
│   OUT    │─────────────────▶│  GPIO38      │ (SSA1)
│          │   (数字信号)     │              │
│   GND    │────────────────▶│  GND         │
│          │                 │              │
└──────────┘                 └──────────────┘

说明：
- GPIO38 对应扩展板接口 SSA1
- GPIO38 是仅输入引脚（Input Only）
- 输出逻辑：1=检测到障碍物，0=无障碍物
- 检测距离：2-30cm（可通过电位器调节）
- 内部启用上拉电阻
```

### 3.6 交通灯控制模块

```
交通灯模块                   ESP32-S3 核心板
┌──────────┐                 ┌──────────────┐
│          │                 │              │
│   VCC    │────────────────▶│  3.3V/5V     │
│          │                 │              │

│          │   (控制信号2)    │              │

```

```
│          │                 │              │
│   VCC    │────────────────▶│  5V          │
│          │                 │              │
```

```
│          │                 │  (通过485转换)│
│   B-     │◀───────────────▶│  UART0_RX    │ (GPIO44)
│          │                 │  (通过485转换)│
│   GND    │────────────────▶│  GND         │
│          │                 │              │
└──────────┘                 └──────────────┘
                              │              │
                              │  GPIO19      │ (RS485方向控制)

说明：
- UART0 (GPIO43/44) 用于RS485通信
- GPIO19 控制RS485收发方向（1=发送，0=接收）
- 波特率：115200
- AI摄像头使用自定义协议通信
```

## 7. 引脚冲突说明

### 7.1 ADC2 与 WiFi 冲突

```
⚠️ 冲突组1：ADC2 通道
┌─────────────────────────────────────┐
│ GPIO18 (ADC2_CH7) - 灰度传感器左     │
│ GPIO20 (ADC2_CH9) - 灰度传感器右     │
└─────────────────────────────────────┘
         │
         ▼
    与 WiFi 互斥
    
解决方案：
1. 不使用 WiFi 功能
2. 改用 ADC1 通道（GPIO47/48）
3. 使用五路灰度I2C传感器替代
```

### 7.2 GPIO20 多功能冲突

```
⚠️ 冲突组2：GPIO20
┌─────────────────────────────────────┐
│ 灰度传感器右 (ADC2_CH9)              │
│ I2C0_SDA (PCF8574扩展芯片)           │
│ 五路灰度I2C_SDA                      │
└─────────────────────────────────────┘
         │
         ▼
    三选一使用
    
当前配置：灰度传感器占用
影响：PCF8574 扩展芯片不可用
```

### 7.3 GPIO39/40 多功能冲突

```
⚠️ 冲突组3：GPIO39/40
┌─────────────────────────────────────┐
│ 交通灯控制 (SIGNAL1/SIGNAL2)         │
│ MPU6050 (I2C1_SDA/SCL)               │
│ VL53L0X (I2C1_SDA/SCL)               │
└─────────────────────────────────────┘
         │
         ▼
    三选一使用
    
当前配置：交通灯控制
```

## 8. 推荐引脚配置方案

### 方案A：WiFi + 循迹（推荐）

```
✅ 使用五路灰度I2C传感器
   - GPIO18: I2C1_SCL
   - GPIO20: I2C1_SDA
   - 支持WiFi功能
   - 更高精度的循迹

✅ 使用交通灯控制
   - GPIO39/40: 交通灯信号

✅ 使用红外避障
   - GPIO38: 红外传感器

✅ 预留超声波接口
   - GPIO47/48: HC-SR04
```

### 方案B：无WiFi + 二路灰度

```
✅ 使用二路ADC灰度传感器
   - GPIO18: 左灰度 (ADC2_CH7)
   - GPIO20: 右灰度 (ADC2_CH9)
   - 不支持WiFi

✅ 使用交通灯控制
   - GPIO39/40: 交通灯信号

✅ 使用红外避障
   - GPIO38: 红外传感器

✅ 使用超声波测距
   - GPIO47/48: HC-SR04
```

## 9. 接口扩展说明

### 9.1 可用的预留引脚

| GPIO | 特性 | 推荐用途 |
|------|------|---------|
| GPIO47 | ADC1_CH6 | 模拟传感器、超声波ECHO |
| GPIO48 | ADC1_CH7 | 模拟传感器、超声波TRIG |
| GPIO1 | 通用IO | 数字传感器 |

### 9.2 不可用的引脚

| GPIO | 原因 |
|------|------|
| GPIO6-11 | 连接到SPI Flash，不能使用 |
| GPIO19 | RS485方向控制，已占用 |
| GPIO43-44 | UART0，用于日志和RS485 |

## 10. 硬件调试建议

### 10.1 引脚测试顺序

1. 首先测试电源和地线连接
2. 测试LED和蜂鸣器（简单输出）
3. 测试按键和红外避障（简单输入）
4. 测试灰度传感器（ADC输入）
5. 测试通信接口（UART、I2C）
6. 测试电机和编码器（PWM和PCNT）

### 10.2 常用测试工具

- 万用表：测量电压和连通性
- 示波器：查看信号波形和时序
- 逻辑分析仪：分析数字通信协议
- 串口调试助手：测试UART通信
