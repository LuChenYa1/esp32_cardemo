# 五路灰度传感器测试程序说明

## 概述

`five_way_gray_test.c` 是五路灰度传感器的完整测试程序，用于验证传感器功能是否正常。

## 测试项目

### 测试1：基本读取测试
- 测试传感器是否能正常读取数据
- 验证单个传感器和批量读取功能

### 测试2：黑线检测测试
- 测试黑线检测功能
- 验证不同阈值的检测效果

### 测试3：实时监控测试（10秒）
- 持续显示传感器值
- 观察传感器对不同表面的响应
- 实时显示黑线/白色状态

### 测试4：统计信息测试
- 显示读取次数、错误次数
- 计算成功率
- 验证后台任务运行状态

### 测试5：校准功能测试
- 显示当前校准参数
- 可选：执行完整校准流程

### 测试6：归一化值测试
- 测试归一化功能
- 显示原始值和归一化值的对应关系

### 测试7：压力测试（5秒）
- 快速连续读取
- 测试系统稳定性
- 验证后台任务性能

## 使用方法

### 1. 编译和烧录

```bash
# 设置测试程序为主程序
cd main
# 备份当前main.c（如果需要）
mv main.c main_backup.c
# 使用测试程序
cp five_way_gray_test.c main.c

# 编译
cd ..
idf.py build

# 烧录
idf.py flash

# 查看输出
idf.py monitor
```

### 2. 硬件连接

确保五路灰度传感器正确连接：
- SCL → GPIO18
- SDA → GPIO20
- VCC → 3.3V/5V
- GND → GND

### 3. 运行测试

烧录后，程序会自动运行所有测试项目，输出类似：

```
========================================
五路灰度传感器测试程序
========================================
初始化板级配置...
初始化五路灰度传感器...
I2C总线初始化成功: SCL=GPIO18, SDA=GPIO20, 地址=0x4F, 频率=400kHz
五路灰度传感器初始化完成，扫描任务已启动（优先级: 6，周期: 2ms）
初始化成功！开始测试...

========================================
测试1: 基本读取测试
========================================
传感器值: [3200, 3150, 3180, 3210, 3190]
传感器0: 3200
传感器1: 3150
...
```

### 4. 校准传感器（可选）

如需校准，在 `test_calibration()` 函数中取消注释：

```c
// 取消注释以下代码进行校准
ESP_LOGI(TAG, "开始校准流程...");
five_way_gray_i2c_calibrate();
```

校准流程：
1. 将传感器放在白色区域，等待5秒
2. 自动采集白色基准值
3. 将传感器放在黑线上，等待5秒
4. 自动采集黑线基准值
5. 保存校准数据

## 预期结果

### 正常情况

- 所有测试通过
- 传感器值在合理范围内（白色：800-1000，黑线：100-400）
- 错误次数为0或极少
- 成功率 > 99%

**数据范围说明：**
- 传感器输出为10位ADC值（0-1023）
- 白色表面通常在800-1000之间
- 黑线通常在100-400之间
- 中间值约为512

### 异常情况

#### 初始化失败

```
五路灰度传感器初始化失败: ESP_FAIL
请检查:
1. 传感器是否正确连接到GPIO18(SCL)和GPIO20(SDA)
2. 传感器I2C地址是否为0x4F
3. 是否与二路ADC灰度传感器冲突
```

**解决方法：**
- 检查硬件连接
- 确认I2C地址（使用I2C扫描工具）
- 确保未同时初始化二路ADC灰度传感器

#### 数据读取失败

```
I2C读取失败 (连续错误: 5): ESP_ERR_TIMEOUT
```

**解决方法：**
- 检查传感器供电
- 缩短I2C总线长度
- 添加外部上拉电阻（4.7kΩ）
- 降低I2C速率（修改为100kHz）

#### 数据未就绪

```
数据未就绪
```

**解决方法：**
- 等待几毫秒后重试
- 检查后台任务是否正常运行

## 故障排除

### 问题1：传感器值全为0

**原因：**
- 传感器未连接或损坏
- I2C通信失败

**解决：**
- 检查硬件连接
- 使用万用表测试传感器供电
- 尝试更换传感器

### 问题2：传感器值不变化

**原因：**
- 传感器固定在某个位置
- 传感器损坏

**解决：**
- 移动传感器到不同表面
- 检查传感器是否正常工作

### 问题3：错误率高

**原因：**
- I2C总线干扰
- 速率过高
- 线路过长

**解决：**
- 缩短I2C总线
- 降低I2C速率到100kHz
- 添加外部上拉电阻

### 问题4：与二路灰度冲突

**原因：**
- 同时初始化了两个模块

**解决：**
- 只初始化其中一个模块
- 检查main.c中的初始化代码
