# TM1640 LED点阵屏测试程序使用说明

## 概述

`main_tm1640_test.c` 是TM1640 LED点阵屏模块的完整测试程序，包含10个测试项目，验证所有功能。

## 硬件连接

| 信号线 | GPIO | 接口位置 | 说明 |
|--------|------|---------|------|
| CLK（时钟） | GPIO34 | SSA3 | 时钟信号线 |
| DIN（数据） | GPIO37 | SSA2 | 数据信号线 |
| VCC | 5V | - | 电源正极 |
| GND | GND | - | 电源负极 |

## 编译和运行

### 方法1：修改CMakeLists.txt（推荐）

编辑 `main/CMakeLists.txt`，将主文件改为测试文件：

```cmake
idf_component_register(
    SRCS "main_tm1640_test.c"
    INCLUDE_DIRS "."
)
```

### 方法2：临时重命名

```bash
# 备份原main.c
mv main/main.c main/main.c.bak

# 将测试文件重命名为main.c
cp main/main_tm1640_test.c main/main.c

# 编译烧录
idf.py build flash monitor
```

### 方法3：使用idf.py直接指定

某些ESP-IDF版本支持：

```bash
idf.py -DMAIN_SRCS="main_tm1640_test.c" build flash monitor
```

## 故障排查

### 问题1：点阵屏无显示

**可能原因：**
- GPIO连接错误
- 电源未接通
- TM1640芯片损坏

**解决方法：**
1. 检查GPIO34和GPIO37连接
2. 确认5V电源供电正常
3. 检查GND是否共地
4. 查看串口日志是否有初始化错误

### 问题2：显示异常或闪烁

**可能原因：**
- 电源不稳定
- 信号线接触不良
- 时序问题

**解决方法：**
1. 使用稳定的5V电源
2. 检查杜邦线连接
3. 缩短连接线长度
4. 添加电源滤波电容

### 问题3：亮度太暗或太亮

**可能原因：**
- 亮度设置不当
- 电源电压不足

**解决方法：**
1. 修改代码中的亮度等级（0-7）
2. 检查电源电压是否为5V
3. 测试不同亮度等级

### 问题4：GPIO冲突

**错误日志：**
```
E (1234) TM1640: GPIO34注册失败，可能与其他模块冲突
```

**解决方法：**
1. 检查是否同时初始化了TM1637模块（共用引脚）
2. 查看pin_definitions.h确认引脚分配
3. 确保没有其他模块使用GPIO34/37

### 问题5：编译错误

**错误信息：**
```
fatal error: tm1640.h: No such file or directory
```

**解决方法：**
1. 确认components/tm1640目录存在
2. 检查CMakeLists.txt配置
3. 清理构建：`idf.py fullclean`
4. 重新构建：`idf.py build`

## 性能指标

- 初始化时间：< 20ms
- 单次刷新时间：1-2ms
- 最大刷新频率：约500Hz
- 内存占用：< 1KB
- 栈空间占用：< 2KB

## 注意事项

1. **引脚复用**：GPIO34/37与TM1637数码管共用，不能同时使用
2. **电源要求**：TM1640需要5V供电，ESP32为3.3V逻辑电平（兼容）
3. **刷新机制**：修改显示缓存后必须调用 `tm1640_refresh()`
4. **亮度调节**：建议使用5-7级亮度以获得较好显示效果
5. **长时间运行**：测试程序会循环运行，可随时按Ctrl+C停止

## 扩展测试

如需添加自定义测试，可在 `app_main()` 中添加新的测试函数：

```c
static void test_custom_pattern(void)
{
    ESP_LOGI(TAG, "【自定义测试】我的图案");
    
    tm1640_clear();
    
    // 在这里添加你的绘图代码
    // ...
    
    tm1640_refresh();
    vTaskDelay(pdMS_TO_TICKS(2000));
}

void app_main(void)
{
    // ... 现有代码 ...
    
    test_custom_pattern();  // 添加自定义测试
    
    // ... 现有代码 ...
}
```
